# React Landing Page Prompt

Выступай как senior frontend developer и UX/UI designer, специализирующийся на современных высококонверсионных лендингах.

Создай production-ready лендинг на React + TypeScript + Vite + SCSS Modules.

## Требования к архитектуре

- Используй чистую и масштабируемую архитектуру
- Каждый экран должен быть отдельной переиспользуемой секцией
- Компоненты должны быть маленькими и читаемыми
- Не допускать переусложнения
- Избегать дублирования кода
- Использовать semantic HTML
- Использовать понятную структуру проекта
- Использовать переиспользуемые UI-компоненты

## Требования к UI/UX

- Современный минималистичный дизайн
- Премиальный и чистый визуальный стиль
- Крупная читаемая типографика
- Сильная визуальная иерархия
- Много воздуха между элементами
- Responsive mobile-first layout
- Единая система отступов
- Мягкие тени
- Плавные скругления
- Современные кнопки и карточки
- Чистые секции без визуального шума
- Один экран = одна основная мысль

## Правила дизайна

- Использовать max-width контейнер
- Соблюдать одинаковые paddings и margins
- Использовать максимум 2 шрифта
- Использовать минимальную цветовую палитру
- Аккуратно использовать градиенты
- Избегать визуального мусора
- Не использовать слишком много цветов
- Не делать агрессивные анимации
- Не использовать устаревшие UI-паттерны

## Адаптив

- Mobile-first подход
- Полная адаптивность под все устройства
- Плавное адаптивное поведение
- Корректные отступы на мобильных устройствах
- Не допускать поломки layout
- Не использовать absolute positioning без необходимости

## Анимации

- Использовать плавные минималистичные анимации
- Использовать Framer Motion только при необходимости
- Анимации должны быть легкими
- Не перегружать интерфейс эффектами
- Делать упор на premium-feel

## Качество кода

- Код должен быть чистым и поддерживаемым
- Использовать TypeScript корректно
- Использовать переиспользуемые константы и переменные
- Не использовать inline styles
- Не создавать огромные JSX-компоненты
- Не использовать magic numbers
- Код должен быть читаемым
- Добавлять комментарии только при необходимости

## Производительность

- Оптимизировать сайт под быструю загрузку
- Использовать lazy loading для изображений
- Использовать WebP/AVIF
- Не подключать тяжелые библиотеки без необходимости
- Оптимизировать рендеринг
- Использовать code splitting при необходимости

## SEO

- SEO-friendly структура
- Semantic HTML tags
- Только один H1
- Правильная иерархия H2/H3
- Добавить meta title и meta description
- Добавить OpenGraph теги
- Использовать alt у изображений
- Оптимизировать читаемость текста
- Оптимизировать контент под поисковые системы и AI search

## Контент

- Текст должен быть коротким и легко читаемым
- Избегать AI-style воды
- CTA-блоки должны быть понятными
- Добавлять элементы доверия
- Делать упор на читаемость и конверсию
- Использовать современные маркетинговые подходы

## Важно

- Не использовать inline CSS
- Не создавать огромные компоненты
- Не делать сложную вложенность
- Не использовать случайные размеры и цвета
- Не перегружать интерфейс
- Сохранять чистую и масштабируемую структуру проекта

## Результат

Сгенерируй полноценный production-ready frontend-код с чистой архитектурой, адаптивным layout, современным UI, хорошим UX и структурой уровня профессионального коммерческого лендинга.
