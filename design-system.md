# Design System: Русский зодчий

Документ является основным визуальным и интерфейсным контрактом проекта. Перед созданием новых секций, компонентов, SCSS-модулей, страниц Astro/React и PHP-интеграций нужно сверяться с ним.

## 1. Общее направление

Стиль сайта: современный, чистый, уверенный, архитектурный.

Сайт не должен выглядеть как перегруженный luxury или "дорого-богато". Нужен спокойный, профессиональный образ компании: много воздуха, крупные фотографии, ясная структура, белые секции и фирменные темно-зеленые акценты.

Основная ассоциация: современный сайт архитектурно-строительной компании с доверием, опытом, понятным процессом и удобной заявкой через Telegram-бота.

Ключевая формула:

```text
G24-like modern renovation website + фирменный зеленый стиль + белые секции + крупные фото + простая сетка + минимум декоративности
```

## 2. Принципы интерфейса

- Один экран раскрывает одну основную мысль.
- Белый фон является базовым фоном проекта.
- Темно-зеленый используется для смысловых фирменных секций, шапки, CTA и футера.
- Золотой используется только как теплый акцент: метки, кнопки, тонкие детали.
- Фотографии должны быть крупными, спокойными и предметными: архитектура, фасады, интерьерные детали, процесс, материалы, реализованные объекты.
- Декоративность минимальная. Никаких лишних паттернов, шумных градиентов, перегруженных иконок и тяжелых теней.
- Визуальный ритм строится на сетке, типографике, воздухе и качественных изображениях.
- Интерфейс должен помогать заявке: пользователь быстро понимает услугу, доверяет компании и видит понятный путь в Telegram-бот.

## 3. Цветовая система

Цвета задаются как CSS custom properties в глобальном SCSS-файле.

```scss
:root {
  // Brand
  --color-green-950: #102820;
  --color-green-900: #17352c;
  --color-green-800: #204339;

  --color-gold-500: #d9a15f;
  --color-gold-400: #e6b777;

  // Light base
  --color-white: #ffffff;
  --color-cream: #f8f5ef;
  --color-light: #fbfaf7;

  // Text
  --color-text: #17231f;
  --color-muted: #6f7773;
  --color-light-text: #f7f3ea;

  // Borders
  --color-border: #e8e2d8;
  --color-border-dark: rgba(255, 255, 255, 0.16);
}
```

### Правила применения

- `--color-white` - основной фон большинства секций.
- `--color-cream` - мягкая подложка для второстепенных блоков, если белого недостаточно.
- `--color-green-900` - основной темный фирменный фон.
- `--color-green-950` - самый глубокий цвет для заголовков, логотипа, важных текстовых акцентов.
- `--color-gold-500` - акцент для CTA, label, тонких деталей.
- `--color-muted` - вторичный текст.

Не использовать палитру как набор декоративных цветов. На одном экране достаточно белого, текста, одного зеленого и редкого золотого акцента.

## 5. Типографика

Использовать максимум два шрифта:

```scss
:root {
  --font-heading: "Cormorant Garamond", "Georgia", serif;
  --font-body: "Inter", "Manrope", "Arial", sans-serif;
}
```

Заголовки должны быть крупными, спокойными и архитектурными, без чрезмерной декоративности.

```scss
h1,
h2,
h3 {
  font-family: var(--font-heading);
  font-weight: 500;
  letter-spacing: -0.02em;
}

body {
  font-family: var(--font-body);
  color: var(--color-text);
}
```

### Рекомендуемая шкала

```scss
:root {
  --text-xs: 12px;
  --text-sm: 14px;
  --text-md: 16px;
  --text-lg: 18px;
  --text-xl: 22px;

  --heading-sm: clamp(28px, 3vw, 40px);
  --heading-md: clamp(36px, 4vw, 56px);
  --heading-lg: clamp(42px, 5vw, 72px);
}
```

Правила:

- На странице должен быть только один `h1`.
- Заголовки не должны быть слишком плотными или кричащими.
- Основной текст короткий, предметный, без AI-style воды.
- Не использовать слишком мелкий текст в ключевых CTA и формах.

```scss
:root {

  --container-wide: 1360px;

  --space-2: 8px;
  --space-3: 12px;
  --space-4: 16px;
  --space-5: 20px;
  --space-6: 24px;
  --space-8: 32px;
  --space-10: 40px;
  --space-12: 48px;
  --space-14: 56px;
  --space-16: 64px;
  --space-20: 80px;
  --space-24: 96px;
}

.container {
  width: min(100% - 32px, var(--container));
  margin-inline: auto;
}
```

Секции:

```scss
.section {
  padding-block: clamp(72px, 9vw, 128px);
}
```

Правила:

- Mobile-first.
- Не использовать случайные размеры.
- Не использовать `absolute` для основной раскладки без необходимости.
- Layout должен сохранять структуру на мобильных, планшетах и десктопе.
- Между секциями должно быть достаточно воздуха, но без ощущения пустоты.

## 7. Hero

Hero светлый, современный и уверенный. Он должен сразу показывать бренд, услугу и сильный визуальный сигнал через крупную фотографию.

```scss
.hero {
  background: var(--color-white);
  overflow: hidden;
}

.hero__inner {
  min-height: 620px;
  display: grid;
  grid-template-columns: 0.9fr 1.3fr;
  align-items: center;
  gap: 56px;
}

.hero__label {
  margin-bottom: 20px;
  color: var(--color-gold-500);
  font-size: 12px;
  font-weight: 600;
  letter-spacing: 0.16em;
  text-transform: uppercase;
}

.hero__title {
  max-width: 620px;
  margin-bottom: 24px;
  font-family: var(--font-heading);
  font-size: var(--heading-lg);
  line-height: 1.05;
  font-weight: 500;
  color: var(--color-green-950);
}

.hero__text {
  max-width: 420px;
  margin-bottom: 32px;
  color: var(--color-muted);
}

.hero__image {
  height: 100%;
  min-height: 620px;
}

.hero__image img {
  width: 100%;
  height: 100%;
  object-fit: cover;
}
```

На мобильных hero должен перестраиваться в одну колонку: сначала текст и CTA, затем изображение. Крупное изображение сохраняется, но не ломает первый экран.

## 10. Изображения

Изображения являются главным визуальным активом сайта.

Требования:

- Использовать реальные или реалистичные фотографии архитектуры, отделки, фасадов, интерьеров, деталей материалов и процесса.
- Не использовать абстрактные 3D-иллюстрации вместо объектов компании.
- Не затемнять фотографии чрезмерными overlay.
- Не использовать темные, размытые, случайные stock-like изображения.
- Для карточек проектов использовать стабильное соотношение сторон.
- Для hero использовать крупный предметный кадр.
- Добавлять осмысленный `alt`.
- По возможности хранить изображения в WebP/AVIF.

## 11. Секции лендинга

Рекомендуемая структура:

1. Header
   - Логотип/название.
   - Навигация.
   - CTA в Telegram.
   - Темно-зеленый фон.

2. Hero
   - Светлый фон.
   - Короткий label.
   - Один сильный `h1`.
   - Краткое описание.
   - Основной CTA в Telegram-бот.
   - Вторичный CTA к проектам/услугам.
   - Крупная фотография.

3. Advantages
   - Темно-зеленая секция.
   - 3-4 понятных преимущества.
   - Без перегруза и длинных текстов.

4. Services
   - Белый фон.
   - Карточки услуг.
   - Короткие описания и понятные формулировки.

5. Projects
   - Белый или cream фон.
   - Крупные карточки реализованных объектов.
   - Фокус на фото и конкретике.

6. Process
   - Белый фон.
   - Простые этапы работы.
   - Акцент на прозрачности сроков, стоимости и коммуникации.

7. Reviews / Trust
   - Темно-зеленая секция.
   - Белые карточки отзывов или блок доверия.
   - Не перегружать количеством.

8. Telegram CTA / Form
   - Темно-зеленый фон.
   - Понятный сценарий: оставить заявку, получить расчет, задать вопрос.
   - Можно использовать форму с PHP-обработчиком и уведомлением в Telegram.

9. Footer
   - Темно-зеленый фон.
   - Контакты, навигация, реквизиты/правовая информация при необходимости.

## 12. Telegram-бот и заявки

Telegram должен восприниматься как основной быстрый канал действия, а не случайная ссылка.

CTA-формулировки:

- `Получить расчет в Telegram`
- `Обсудить проект`
- `Оставить заявку`
- `Перейти в Telegram-бот`

UX-правила:

- Главный CTA повторяется в hero, середине страницы и финальном CTA.
- Если есть форма, она должна быть короткой: имя, телефон/Telegram, тип задачи, комментарий.
- После отправки формы показывать понятное состояние успеха.
- Ошибки формы писать человеческим языком.
- PHP-обработчик не должен раскрывать токены Telegram-бота на клиенте.

## 13. Анимации

Анимации нужны только для premium-feel, а не для демонстрации эффекта ради эффекта.

Разрешено:

- мягкое появление секций при скролле;
- легкий hover у карточек;
- аккуратный hover у кнопок;
- плавное раскрытие мобильного меню;
- небольшая смена состояния формы.

Не использовать:

- агрессивные параллаксы;
- тяжелые постоянные движения;
- мигающие акценты;
- анимации, мешающие читать текст;
- layout shift при загрузке.

Всегда учитывать `prefers-reduced-motion`.

## 14. Контентный тон

Тон: спокойный, уверенный, конкретный.

Писать:

- коротко;
- предметно;
- без канцелярита;
- без обещаний, которые нельзя подтвердить;
- с фокусом на доверие, процесс, сроки, качество и удобную коммуникацию.

Избегать:

- "уникальные решения для каждого клиента" без конкретики;
- "лучшие на рынке";
- длинных абзацев;
- AI-style воды;
- чрезмерного luxury-языка.

Пример хорошей формулировки:

```text
Проектируем и реализуем частные дома, фасады и интерьерные решения с понятным процессом, прозрачной сметой и постоянной связью в Telegram.
```

## 15. SEO и семантика

Требования:

- Один `h1` на странице.
- Секции размечать через `section`, `header`, `main`, `footer`.
- У каждой крупной секции должен быть понятный `h2`.
- Добавить meta title, description и OpenGraph.
- У всех значимых изображений должен быть `alt`.
- Навигация должна использовать `nav`.
- Кнопки-ссылки на Telegram оформлять как ссылки, действия формы - как `button`.
- Контент должен быть понятен людям и AI search: кто компания, что делает, где помогает, как оставить заявку.

## 16. Кодовые правила

- React-компоненты маленькие и читаемые.
- Каждая крупная секция - отдельный компонент.
- Стили секций - через SCSS Modules.
- Глобальные токены и базовые стили - в общем SCSS-файле.
- Без inline styles.
- Без случайных magic numbers: размеры выносить в токены или локальные понятные переменные.
- Не подключать тяжелые библиотеки без необходимости.
- Framer Motion использовать только если анимация действительно нужна.
- PHP использовать для серверной логики: обработка формы, отправка уведомлений, интеграция с Telegram Bot API.

## 17. Accessibility

- Контраст текста должен быть достаточным на светлом и темном фоне.
- Все интерактивные элементы должны иметь видимое focus-состояние.
- Кнопки и ссылки должны иметь понятный текст.
- Не полагаться только на цвет для передачи смысла.
- Мобильные tap targets не меньше 44px по высоте.
- Мобильное меню должно быть доступно с клавиатуры.
- Форма должна иметь label для каждого поля.

## 18. Чеклист перед добавлением новой секции

Перед реализацией секции проверить:

- Секция решает одну ясную задачу?
- Фон соответствует общей логике светлых и темно-зеленых секций?
- Есть достаточно воздуха?
- Заголовок короткий и конкретный?
- Текст можно сократить без потери смысла?
- Нет лишних карточек, иконок, цветов или декоративных элементов?
- CTA ведет к понятному действию?
- Секция нормально работает на мобильных?
- Все изображения имеют правильный кроп, размер и `alt`?
- Компонент можно переиспользовать или поддерживать без боли?
